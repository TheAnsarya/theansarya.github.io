1. Create a GitHub repository and enable Pages  
Create a public repo (or use an existing one). Add an index.html (even a one‑line page). In the repo Settings → Pages, select the branch (usually main) and root folder, then save. GitHub will publish at https://<username>.github.io/<repo>.

2. Add a CNAME file for your custom domain  
In the repo root add a file named CNAME containing your domain (e.g., example.com). Commit and push. GitHub uses this to associate the repo with your domain.

3. Sign up for Cloudflare and add your domain  
Create a free Cloudflare account, add your domain, and let Cloudflare scan your DNS. Cloudflare will give you nameservers to set at your registrar; update them there and wait for propagation.

4. Configure DNS records in Cloudflare  
In Cloudflare DNS:

Add a CNAME record for www pointing to <username>.github.io.

Add an A record for the root (@) pointing to GitHub Pages IPs if required, or create a CNAME flattening to www (Cloudflare supports this). Ensure the Cloudflare proxy (orange cloud) is enabled so Cloudflare serves HTTPS.

5. Enforce HTTPS and wait for certificate issuance  
In Cloudflare SSL/TLS settings, choose Full (strict) if you later add an origin cert, or Flexible temporarily. Enable Always Use HTTPS and Automatic HTTPS Rewrites. Cloudflare issues Universal SSL automatically; this can take minutes to a few hours.

6. Verify the site and finish Marketplace verification  
Visit https://yourdomain to confirm HTTPS. Then retry the Visual Studio Marketplace verification flow — if it requires a TXT record, add that in Cloudflare DNS and verify there as well.
