üå∫ Disassemble a ROM with Peony

  1. Clone the Peony repository and build with dotnet build
  2. Run peony disasm game.nes --output src/
  3. Peony generates annotated assembly files and a Pansy metadata file
  4. Open the output in VS Code with Poppy extension for syntax highlighting
  5. Edit labels, add comments, and refine the disassembly
Read full guide ‚Üí

üå∏ Build a ROM with Poppy

  1. Create a main.pasm file with your assembly code
  2. Add .platform nes or .platform snes directive
  3. Run poppy build main.pasm --output game.nes
  4. Poppy validates, assembles, and generates the ROM
  5. Test in Mesen2 or your favorite emulator
Read full guide ‚Üí

üåº Share Metadata with Pansy

  1. Export labels from Mesen2 using the Pansy export feature
  2. Or create a Pansy file programmatically with the C# library
  3. Include symbols, comments, cross-refs, and memory regions
  4. Share the .pansy file with collaborators
  5. Import into Peony, Poppy, or other compatible tools
Read full guide ‚Üí

üîÑ Complete Roundtrip Workflow

  1. Start with an original ROM and disassemble with Peony
  2. Store all analysis data in a Pansy metadata file
  3. Edit the source code ‚Äî fix bugs, add features, translate text
  4. Reassemble with Poppy to create a modified ROM
  5. Verify byte-identical output (for vanilla) or test your changes
